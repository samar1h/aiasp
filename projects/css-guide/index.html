01Centering Children

The one trick you need: put display: flex on the parent, then use align-items and justify-content.

Center everything (both axes)

.parent {
  display: flex;
  align-items: center;      /* vertical center */
  justify-content: center;  /* horizontal center */
  height: 200px;            /* parent needs a height */
}
Live Demo — centered child
I am centered 🎯

Flex direction: row vs column

/* Default: children go LEFT → RIGHT (row) */
.row-parent { display: flex; flex-direction: row; }

/* Stack children TOP → BOTTOM */
.col-parent { display: flex; flex-direction: column; }
PropertyEffect
justify-content: flex-startpack to start (default)
justify-content: flex-endpack to end
justify-content: centercenter along main axis
justify-content: space-betweenspace between children, no outer gap
justify-content: space-aroundequal space around each child
align-items: centercenter along cross axis
align-items: stretchfill cross axis (default)
gap: 16pxspace between children
💡 Mental model: justify-content works along the direction the items flow. align-items works perpendicular to it. In a row: justify = horizontal, align = vertical.

02Side-by-Side Splits (20% / 80%)

Two ways to do it. Flexbox is great for fluid splits. Grid is better when you want strict column ratios.

Method A — Flexbox with width

.container {
  display: flex;
  gap: 12px;
}
.sidebar  { width: 20%; flex-shrink: 0; }  /* fixed % */
.content  { flex: 1; }               /* take the rest */
Live Demo — flex 20/80
20%
80% (flex: 1)

Method B — CSS Grid (cleaner ratios)

.container {
  display: grid;
  grid-template-columns: 1fr 4fr;  /* 1:4 = 20% / 80% */
  gap: 12px;
}
/* children go into columns automatically */
/* Other useful ratios: */
grid-template-columns: 1fr 3fr;      /* 25% / 75% */
grid-template-columns: 1fr 1fr;      /* 50% / 50% */
grid-template-columns: 1fr 1fr 1fr;  /* 3 equal columns */
grid-template-columns: repeat(4, 1fr); /* 4 equal columns */
Live Demo — grid 1fr 3fr
1fr
3fr
📌 flex: 1 means "take all remaining space". If two children both have flex: 1, they split space equally. flex: 2 takes twice as much as flex: 1.

Wrapping children (cards, tags)

.card-grid {
  display: flex;
  flex-wrap: wrap;   /* overflow to next line */
  gap: 12px;
}
.card {
  flex: 1 1 200px;   /* grow, shrink, min-width 200px */
}

03Auto-Scale — No Hardcoded Sizes

Stop writing font-size: 32px and padding: 60px. Here are the tools that adapt automatically.

clamp() — the most useful function

Syntax: clamp(minimum, ideal, maximum)

h1 {
  font-size: clamp(20px, 5vw, 48px);
  /* min 20px, grows with viewport, max 48px */
}
.container {
  padding: clamp(16px, 4vw, 60px);
  max-width: 900px;
  margin: 0 auto;   /* center horizontally */
}

Relative units

UnitRelative toUse for
%parent elementwidths, heights of children
vwviewport widthfull-width sections, fluid type
vhviewport heightfull-screen sections

emcurrent element's font-sizerelative spacing near text
dvhdynamic viewport (mobile-aware)full-screen on mobile

Media queries — change layout at breakpoints

/* Default styles: mobile first */
.container { flex-direction: column; }

/* On wider screens: switch to row */
@media (min-width: 768px) {
  .container { flex-direction: row; }
}

/* Common breakpoints: */
/* 480px  — large phones  */
/* 768px  — tablets       */
/* 1024px — small laptops */
/* 1280px — desktops      */

Auto-fit grid (cards that reflow)

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  /* as many columns as fit, each at least 200px wide */
  gap: 16px;
}
💡 This single line replaces most media query grid code. Cards automatically reflow from 1 to 2 to 3+ columns as the screen grows.

04Mobile Toolbar — Above Keyboard, Not Floating

On mobile, you want a bar that stays at the bottom of the screen but moves up when the keyboard appears. This is not position: fixed alone — you need env(safe-area-inset-bottom) too.

The full pattern

/* In your HTML */
<meta name="viewport" content="width=device-width,
      initial-scale=1, interactive-widget=resizes-content">

/* In your CSS */
.toolbar {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  background: var(--surface);
  border-top: 1px solid var(--border);
  padding: 10px 16px;
  /* ↓ adds space for iPhone home indicator */
  padding-bottom: calc(10px + env(safe-area-inset-bottom));
  z-index: 100;
}

/* Your page body needs bottom padding so toolbar doesn't cover content */
body { padding-bottom: 80px; }
⚠️ The viewport meta trick: Adding interactive-widget=resizes-content tells the browser to shrink the viewport when the keyboard opens — so your fixed toolbar naturally moves up with the keyboard. Without it, the toolbar stays behind the keyboard.

For an input + send button inside the toolbar

.toolbar {
  display: flex;
  gap: 8px;
  align-items: center;
}
.toolbar input {
  flex: 1;          /* takes all space except button */
  padding: 10px;
  border-radius: 8px;
  border: 1px solid var(--border);
  background: var(--bg);
  color: var(--text);
  font-size: 16px;  /* prevents iOS zoom-on-focus */
}
.toolbar button {
  flex-shrink: 0;
  padding: 10px 16px;
}

05Themes & Accents — CSS Variables + JS

CSS custom properties (variables) are the foundation of any theming system. They live on :root (the whole page) and can be changed instantly with one line of JS.

Define variables in CSS

:root {
  --bg:      #0f0f11;
  --surface: #1a1a1f;
  --text:    #e8e6e3;
  --accent:  #f97316;   /* change this to retheme everything */
}

/* Use them everywhere */
body        { background: var(--bg); color: var(--text); }
button      { background: var(--accent); }
.card       { background: var(--surface); }

Toggle dark/light with a class

/* CSS — define overrides */
.light {
  --bg:      #f5f4f0;
  --surface: #ffffff;
  --text:    #1a1a1a;
}

/* JS — toggle the class on body */
function toggleTheme() {
  document.body.classList.toggle('light');
}

Change accent color via JS

/* Change --accent to anything at runtime */
function setAccent(color) {
  document.documentElement.style
    .setProperty('--accent', color);
}

/* Examples */
setAccent('#f97316');   /* orange */
setAccent('#7c3aed');   /* purple */
setAccent('#22c55e');   /* green  */
💡 document.documentElement is the <html> element — same as :root in CSS. Setting a property there overrides everything below it.

06Units Cheatsheet

/* LENGTHS */
width: 100%          → full parent width
width: 50vw         → half screen width
height: 100dvh      → full screen height (mobile-safe)
padding: 1rem       → 16px (assuming 16px root)
gap: 0.5em          → half the element's font-size

/* SIZING SHORTCUTS */
width: min-content  → shrink to fit content
width: max-content  → expand to fit content, no wrap
width: fit-content  → max-content but respect parent

/* CENTERING A BLOCK */
max-width: 800px;
margin: 0 auto      → centers horizontally

07Positioning

ValueBehaviour
staticDefault. Normal document flow, top/left ignored.
relativeNormal flow, but you can nudge it with top/left/etc. Also creates a positioning context for children.
absoluteRemoved from flow. Positioned relative to nearest non-static ancestor.
fixedRemoved from flow. Positioned relative to the viewport. Stays on screen while scrolling.
stickyNormal flow until it hits the scroll position you set, then sticks. Great for headers/sidebars.
/* Typical overlay pattern */
.card    { position: relative; }
.badge   {
  position: absolute;
  top: -8px;
  right: -8px;
}

/* Sticky nav */
nav {
  position: sticky;
  top: 0;
  z-index: 50;
}

08Overflow & Scroll

/* Hide overflow content */
overflow: hidden;

/* Always show scrollbar */
overflow: scroll;

/* Scrollbar only when needed (most common) */
overflow: auto;

/* Control axes independently */
overflow-x: auto;   /* horizontal scroll */
overflow-y: hidden; /* clip vertical */

/* Scrollable panel — common pattern */
.panel {
  height: 300px;     /* must have defined height */
  overflow-y: auto;
}

/* Truncate text with ellipsis */
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
  max-width: 200px;
}

09Quick Tips

/* Smooth transitions when anything changes */
* { transition: background 0.2s, color 0.2s; }

/* Prevent images from overflowing */
img { max-width: 100%; height: auto; }

/* Remove default button/input styles */
button { all: unset; cursor: pointer; }

/* Smooth scroll for anchor links */
html  { scroll-behavior: smooth; }

/* box-sizing: border-box on everything — do this always */
*, *::before, *::after { box-sizing: border-box; }

/* Visually hidden but accessible (screen readers) */
.sr-only {
  position: absolute;
  width: 1px; height: 1px;
  clip: rect(0,0,0,0);
  overflow: hidden;
}

/* Vertical centering of multi-line text */
.center-text {
  display: flex;
  align-items: center;
  min-height: 3lh;   /* 3 line heights */
}

/* Prevent text selection on buttons/labels */
.no-select { user-select: none; }

/* Aspect ratio — square / 16:9 / etc */
.video { aspect-ratio: 16 / 9; width: 100%; }
🎓 Learning order: Flexbox → CSS Variables → clamp() → Grid → position sticky/fixed. That combo covers ~90% of real app layouts.
